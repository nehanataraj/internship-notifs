# Writing style reference

Bullets below are the candidate's preferred voice. Match this rhythm and level of detail when rewording. Fix grammar; do not copy typos. Do not import facts from here — facts come only from the uploaded resume.

## Tone

Direct, technical, concrete. **Active command voice** — verb-led bullets (Built, Engineered, Created), never passive.

Each bullet follows **what → how → impact**: did **X** using **Y**, which impacted **Z** (one sentence — accomplishment, method/tools, outcome). Use **parallel structure** — consistent verb forms, no dangling gerunds.

When tailoring: keep this voice; weave in JD keywords lightly. Vary opening verbs across bullets — put skills in the how clause ("using Python"), not repeated Skill-verb prefixes.

Also: sincere and goal-oriented when context fits (summary or cover-adjacent lines) — cares about products at scale, full-stack growth, learning from strong teams. Not salesy or essay-long on the resume.

## Bullet examples (voice reference)

- Created an AWS PostgreSQL database using Python scripts and SQL commands to handle 11+ million entries across multiple integrated databases.
- Collaborated with a multi-disciplinary team during early-stage product development to align database architecture decisions with functional requirements.
- Presented and architected a scalable database infrastructure from multiple third-party APIs from scratch to support product growth.

## Words / patterns to avoid

synergy, rockstar, passionate, leveraged, results-driven
passive voice ("were used to", "was built", "was engineered")
tool-first phrasing ("Python-based X and Y were used to…")
repeated skill-verb prefixes ("Python-engineered", "Python-scripted", "SQL-built" on multiple bullets)

## Words / patterns you prefer

Built, created, architected, collaborated — concrete verbs with tools and numbers when available.
